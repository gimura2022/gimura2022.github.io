NAME

cstyle — my C code style

DESCRIPTION

My code style based on linux kernel code style but I changed some things in it for me.

Indetation

I indent my code by 8 spaces. Lines should be shorter than 110 charcters.

Comments

I never use "//", instead I use "/* comment */", for example:

/* some multiline
 * comment */

In code I comment every defention at top level.

Backets on line/after line

In function defentions I write backets after line but on other cases I write backets on line with space.

Switch cases

Cases should be writing at switch level, for example:

switch (c) {
case 'a':
	...
	break;

case 'b':
	...
	break;

case 'c':
	...
	break;
};

Project strucure

All my projects should have main source file (for example "some_utility.c"), LICENSE file with license, Makefile for compiling and installing project, manpage file (for example "some_utility.1") and README file.

NOTES

Not all my projects satisfy this code style, but I try write new projects following this code style.
